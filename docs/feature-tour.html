ImGui-IDE — feature tour · master @ lobotomy/main

A self-hosting IDE, built in its own image

The immediate-mode IDE that develops itself — and mods your games.

A native Windows editor on Dear ImGui 1.92 + SDL3 GPU, tree-sitter and clangd — with a real DLL plugin system that carries a visual Blueprint scripting graph, a live UEVR game bridge, and Unreal Engine tooling. Everything below ships today.

The dockable shell — navigation, tabbed editor with a minimap, and the UEVR Live bridge docked below. Every panel is a tool you can pop out, split, or hide.

01

A real editor core

Tree-sitter symbols, clangd, and a hand-tuned text widget — not a textarea with syntax colors.

Go-to-definition tree-sitter

A background indexer walks the project (and any extra source locations) into a kind-ranked symbol cache for C/C++/C#/Lua and more. Jump to definitions and declarations across files; per-type member completion is refined by clangd where present.

Generate from declaration cppgen

Stub a definition from a header declaration, split header/source, and get UE-aware member autocomplete. Folding with previews, multi-cursor, subword motion, box-select, and two-stroke chord keybinds throughout.

Language-accurate coloring

A backslash-continued C/C++ macro is one directive: the whole #define — every continuation line — stays preprocessor-colored, and a comment or string opened in the body never bleeds past the macro. Ctrl+A selects to the true top-left; Ctrl+Home lands at line 0, column 0.

Formatters & diagnostics

One-key clang-format / stylua / black / rustfmt on the buffer, with the tool's actual error surfaced (line:column) instead of a generic guess. Markdown preview, PDF viewer, and an in-app image viewer for assets.

02

Navigation & project

The tree keeps pace with real repos — Unreal-sized trees included.

  • Sort by Name · Modified (newest) · Size · Type, folders first, persisted.
  • Extra source locations — index folders outside the project root into both the nav tree and the symbol cache.
  • Smarter symbol excludes — Unreal Intermediate/ Binaries/ Saved/ DerivedDataCache/, cmake-build-*, caches; honors your hidden-from-view folders too.
  • Code / project files only — one filter, now keeping images and PDFs in the tree.
  • Multiple instances again — a projectless launch opens its own window; File ▸ New Window (or per-project).

Nav ▸ Filters — sort order and extra roots, both persisted.

03

Git & the cloud

Branch locally, browse remotely — without leaving the editor.

Project ▸ Browse GitHub Repo — read-only, opens files in a tab.

  • Switch Branch from a live list of locals; New Branch inline — a dirty tree makes git refuse, not clobber.
  • Clone from URL straight into the nav, indexing as files land.
  • Networked repo explorer — browse any public (or private, with a PAT) GitHub repo's tree over the API and open files read-only, no clone.
  • The token is session-only — never written to disk.
  • Built on the same WinHTTP path as the in-app updater; all requests run off the UI thread.
04

Blueprint visual scripting

An Unreal-style node graph that compiles to UEVR Lua — a whole DLL plugin, linking none of its code into the core.

Drag from a typed AActor pin → the menu offers that class's SDK members (contextual, never dumped into the global palette). The wire lands on Target, not the nearest bool.

SDK, done right

Imported reflection dumps feed autocomplete and on-demand, per-class member menus — 10k classes never flood the node list. Prefix folding unifies AActor with a dumped Actor.

Safe codegen

Every callback is generation-guarded and pcall-wrapped. Table variables, For Each / Pairs, Set-If-Unset (x = x or …), and Custom Lua nodes compiled as reusable local functions.

UE-style palette

A toggleable right-hand panel lists every node by category and lets you browse the imported SDK class list — click a member to drop it on the canvas. Middle-mouse drag-scrolls the lists, honoring your invert-pan setting.

05

UEVR Live — a bridge into the running game

Run scripts, inspect state, and import an SDK straight from a live VR session.

  • Run in UEVR — generate and inject the Blueprint's Lua into a running game; Reset BP Script silences it via the generation guard.
  • Interactive Globals & Modules — expand any Lua table/object on demand (pairs() + debug.getinfo) and drill arbitrarily deep.
  • Import SDK from Game — dump live objects' classes over the bridge for autocomplete + on-demand exposure. No file, no game rebuild.
  • Every row has an ins button to drop a Custom Lua node reading exactly that value.

Live Lua object inspection — the debug package, in a table.

06

Unreal Engine tooling

Project awareness, code scaffolding, and asset inspection — for a Blueprint or C++ project.

Open a .uasset and it inspects in-app as JSON — no external app.

  • Open .uasset in-app — a plugin claims the binary and shows the header, imports, and a Blueprint summary as colored, foldable JSON (a Save-As export).
  • Interactive descriptor editor — add a plugin dependency or a module to .uproject/.uplugin, picking Type and LoadingPhase from real UE lists.
  • Class & Verse wizards — UCLASS header/source pairs and creative_device scaffolding.
  • Build the editor target (F6), generate a clangd compile DB, launch the editor, resolve engine-header Go to File.
07

The architecture underneath

A lean core that gains whole feature suites by loading a DLL.

In-process plugin ABI

UEVR/Blueprint and Unreal are real loadable DLLs sharing one ImGui context + heap across the boundary. A core build links zero of their code; a runtime toggle enables each.

Builds never block

Plugins load from a per-process shadow copy, and a PRE-LINK step renames a running image aside — so a rebuild never hits LNK1168 while the app is running.

Work off the UI thread

The symbol index, git poll, fold detection (past 5k lines) and large-file loading all run on workers — a 100k-line file opens without stalling a frame. 496 headless assertions keep it green at /W4 /WX.
